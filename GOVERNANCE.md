# Project Governance

Librain is an open source project started during HackUDC. This document explains how decisions are made and how the community can participate.

## Decision-making

We prefer lightweight and transparent decisions:

1. Proposals happen as GitHub issues (feature requests) or PRs
2. Maintainers review for:
   - Alignment with project goals (decision-first workflow)
   - Security & privacy considerations
   - Code quality / maintainability
3. For significant changes, maintainers may request an RFC-style issue before implementation.

## Project principles

- **Decision-first UX:** reduce cognitive friction; surface the next best action
- **Modularity:** extraction and AI logic should be composable and testable
- **User control:** keep data portable; avoid lock-in
- **Safety & privacy by default:** clear retention rules and minimal storage
- **Transparency:** documented AI providers and data handling

## Releases

- We use Semantic Versioning.
- Changelog entries are required for notable changes.

## Community expectations

- Be respectful and constructive.
- Follow the Code of Conduct (`CODE_OF_CONDUCT.md`).
